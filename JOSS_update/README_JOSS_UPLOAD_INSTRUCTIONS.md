# How to use this JOSS update package

This ZIP intentionally does not modify any source-code file.

1. Copy the `paper/` folder into the root of the GitHub repository.
2. Commit and push the new files to GitHub.
3. Check that `paper/paper.md` and `paper/paper.bib` are visible online.
4. Keep the existing source code, README, LICENSE, CITATION, validation files, screenshots, and sample inputs in the repository.
5. Submit the GitHub repository URL to JOSS.

JOSS expects the paper files to live in the same Git-based repository as the software. The Word file is only a human-readable copy; the actual JOSS paper should be `paper/paper.md` with `paper/paper.bib`.
