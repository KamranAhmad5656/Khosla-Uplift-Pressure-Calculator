# JOSS Submission Checklist for Khosla Hydraulic Foundation Designer

This checklist is for the GitHub repository before submitting to JOSS.

## Paper files
- [ ] `paper/paper.md` is present in the repository.
- [ ] `paper/paper.bib` is present in the repository.
- [ ] The paper is between 750 and 1750 words.
- [ ] The paper includes Summary, Statement of need, State of the field, Software design, Research impact statement, AI usage disclosure, Acknowledgements, and References.

## Repository requirements
- [ ] Repository is public and cloneable without registration.
- [ ] Source code is browsable online.
- [ ] Issue tracker is enabled.
- [ ] `LICENSE.txt` contains an OSI-approved license.
- [ ] `README.md` includes installation instructions.
- [ ] `README.md` includes example usage.
- [ ] `CITATION.cff` is present.
- [ ] Validation material is present.
- [ ] Sample input file is present.
- [ ] Screenshots and exported report are present.

## Review readiness
- [ ] A colleague has installed and run the software using the README instructions.
- [ ] Validation cases can be reproduced.
- [ ] The Zenodo DOI points to the same release described in the paper.
- [ ] Both authors have approved the submitted version.
